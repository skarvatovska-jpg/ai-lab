### Структура папки `ai_engine`
У цій папці зберігаються внутрішні компоненти AI-ядра: промпт-логіка, виклики LLM, моделі перетворення поведінкових відповідей та обчислення burnout-score.

### Про AI engine
Це внутрішнє ядро, що готує промпти, викликає LLM, формує поведінковий вектор і рахує burnout-score.

Що робить engine:
- перетворює сирі відповіді анкети у структурований behavior vector (Big5-like);
- генерує сценарії симуляцій залежно від вектора та стрес-факторів ролі;
- оцінює реакції в симуляціях і повертає сумісність + burnout-risk + поради;
- логуватиме LLM-запити/відповіді для подальшого тюнінгу.

Пайплайн:
1) ingest поведінкових відповідей → нормалізація;
2) prompt-engineering для аналізу та генерації сценаріїв;
3) виклик OpenAI API (analysis → scenarios → burnout prediction);
4) агрегація й формування звіту для бота.

Основні модулі:

1) Модуль Обробки Данных

нормалізація відповідей користувача

перетворення анкетних відповідей у формат, який зручно подати моделі

побудова поведінкового вектора(аналог Big5) (Behavior Vector Model)



2) Модуль LLM-Обробки

підготовка системних промтів

виклик OpenAI API (analysis → scenarios → burnout prediction)

отримання моделей сценаріїв

аналіз відповідей користувача у симуляції



3) Модуль Логіки Симуляцій

управління ігровими сценаріями

адаптація квестів залежно від поведінкового вектору

облік кожної реакції користувача

формування динамічного рейтингу сумісності


4) База Даних (SQLite)

Таблиці:

users (id, ім’я, дата старту, рівень залучення)

behavior_vectors (Big5-like metrics + тригери)

simulation_results (рейтинг, burnout_score, оцінки сценаріїв)

logs (LLM запити → для відлагодження)
